## Supported Versions

- **Current:**:white_check_mark:  
This site is actively maintained and receives regular security updates.

## Reporting a Vulnerability

Any and all vulnerabilities found should be reported to [GitHub Security](https://github.com/Genera1Developer/Curse/security)

- **Private:** Submit a private report through GitHub's security tab.
- **Public:** Create a public issue in the [Issues](https://github.com/Genera1Developer/Curse/issues) section, but please mark it as "Security Issue."

## Responsible Disclosure

We ask that researchers follow responsible disclosure practices. This includes:
- Giving us reasonable time to address the issue before publicly disclosing it.
- Providing us with a detailed report of the vulnerability, including steps to reproduce it.
- Refraining from exploiting the vulnerability for personal gain.

content: # Bug Bounty Program

## Overview

We are committed to providing a secure experience for our users. As part of this commitment, we operate a bug bounty program to reward researchers for reporting security vulnerabilities.

## Program Details

- **Scope:** The program covers all aspects of our web application.
- **Rewards:** Rewards will be determined based on the severity of the vulnerability and the quality of the report.
- **Exclusions:** Vulnerabilities that are already known to us or that are publicly disclosed are not eligible for rewards.

## How to Participate

To participate in the program, please follow these steps:

1. Find a security vulnerability in our web application.
2. Submit a private report to our security team.
3. Wait for a response from our security team.

## Responsible Disclosure

We ask that researchers follow responsible disclosure practices. This includes:
- Giving us reasonable time to address the issue before publicly disclosing it.
- Providing us with a detailed report of the vulnerability, including steps to reproduce it.
- Refraining from exploiting the vulnerability for personal gain.
